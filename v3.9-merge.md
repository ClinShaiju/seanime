# v3.9.0 upstream merge — working notes

**Goal:** bring fork up to date with `origin/main` (5rahim v3.9.0, tag `v3.9.0`) while preserving all fork features.

**Method:** single `git merge origin/main` on `main` (not cherry-picks — 33 upstream commits, 308 files). Analyze collision zones first, resolve 30 conflicts file-by-file, then `go build/vet/test` + web build before committing.

## Upstream v3.9 highlights (from changelog + commits)

- **MpvCore** — new libmpv-based built-in player for Denshi via `mpv-prism` (closed-source for now, ships as prebuilt npm tgz from `seanime.app/assets/mpv-prism/0.1.0/`). New backend package `internal/mpvcore/`, handler `handlers/mpvcore.go`, web feature `_features/mpv-core/`. Experimental, opt-in.
- **Electron 42.4.0** with upstream's *own* AC3/EAC3 patches (their mirror `seanime.app/assets/electron/v42.4.0`). Overlaps our custom runtime — but theirs lacks our DTS/TrueHD additions.
- **Faster stream startups** (b09257d9): torbox/realdebrid/alldebrid poll loops → immediate first check + 1.5s ticker + error-retry tolerance. Same idea as our poll-immediately work; ours is a superset (infoCache, fileIdCache prime, batch fan-out).
- **Redesigned playback pill**: autoselect now reports per-candidate status (`updateCandidates`) to the UI. **No cache-based reordering** — quality-over-speed rule not violated.
- **Premiumize** debrid provider (new, no conflict — new file).
- **mediacore refactor** (ae92bc87): removed mediacore types, new `internal/mpvcore/` adapter split.
- UI restyle wave (Settings/Stats, tabs, inputs, modals, splashscreen).
- Misc: goja chromedp APIs, nakama library-update fix, AllDebrid batch streaming fix, HMAC default-secret fix, MPV conf written to file, screenshot dir prompt.

## Decisions

1. **Version:** take upstream `3.9.0 "Kizami"`. Our stale-tab detection keys off version match; next deploy bakes 3.9.0.
2. **Electron runtime:** keep BOTH mechanisms in `electron-build.yml` — upstream's `sync:mpv-prism` step AND our optional `custom_electron_asset` override. **Caveat:** our custom DTS/TrueHD runtime was built for Electron 39.2.7; it is STALE against 42.4.0. Until rebuilt, builds use upstream's 42.4.0 (AC3/EAC3 ok, DTS/TrueHD missing in VideoCore). Long-term: MpvCore (libmpv) decodes DTS/TrueHD natively → custom runtime becomes unnecessary once MpvCore is the player.
3. **Poll loops (torbox etc.):** keep our superset implementation; graft upstream's transient-error retry tolerance where missing.
4. **Autoselect:** keep our comparison/parsing logic; add upstream's ctx threading + `updateCandidates` reporting so the new pill works.
